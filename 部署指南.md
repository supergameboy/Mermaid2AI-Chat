# 部署指南

## 1. 部署概述

本项目采用 MCP服务+Web应用 的部署形态，Node.js 后端提供 MCP 端点和 WebSocket 服务，前端提供 Web 编辑器，推荐通过 VSCode 插件集成使用。

## 2. 前置条件

### 2.1 系统要求
- Node.js 20+
- pnpm 9+
- TRAE（支持 MCP 的 AI IDE）
- VSCode 1.85+（使用 VSCode 插件时）

### 2.2 端口要求
- 14514：MCP 服务端 + WebSocket
- 5173：Web 编辑器开发服务器（Vite，仅开发模式）

## 3. 安装步骤

### 3.1 获取项目代码

```bash
git clone <project-repo>
cd mermaid2ai-chat
```

### 3.2 安装依赖

```bash
pnpm install
```

### 3.3 构建项目

```bash
pnpm build
```

## 4. VSCode 插件（推荐）

VSCode 插件内嵌编辑器画布，打开面板时自动启动 MCP 服务端，无需手动启动 server。

### 4.1 构建插件

```bash
pnpm --filter @mermaid-editor/vscode-extension build
```

### 4.2 启动扩展开发宿主

**方式一：F5 调试启动（推荐）**

1. 将 `vscode-extension-test/launch.json` 和 `vscode-extension-test/tasks.json` 复制到项目根的 `.vscode/` 目录：
   ```powershell
   # Windows (PowerShell)
   New-Item -ItemType Directory -Force -Path .vscode | Out-Null
   Copy-Item vscode-extension-test/launch.json .vscode/launch.json
   Copy-Item vscode-extension-test/tasks.json .vscode/tasks.json

   # Linux/macOS
   mkdir -p .vscode
   cp vscode-extension-test/launch.json .vscode/launch.json
   cp vscode-extension-test/tasks.json .vscode/tasks.json
   ```
2. 在 VSCode 中打开项目根目录
3. 按 `F5`（或运行面板选择"运行 Mermaid 编辑器扩展"）
4. 自动启动扩展开发宿主窗口

**方式二：命令行启动**

```bash
# Windows (PowerShell)
code --extensionDevelopmentPath="$PWD/packages/vscode-extension"

# Linux/macOS
code --extensionDevelopmentPath="$PWD/packages/vscode-extension"
```

### 4.3 打开编辑器

在扩展开发宿主窗口中：
- 按 `Ctrl+Shift+P` → 输入 `Mermaid: 打开 Mermaid 编辑器`
- 或点击活动栏左侧的 Mermaid 图标

插件会自动启动 MCP 服务端（端口 14514），服务端日志输出到 VSCode 的"Mermaid Editor Server"输出面板。

### 4.4 修改代码后重新加载

- **扩展代码修改**：关闭扩展宿主窗口，重新构建（`pnpm --filter @mermaid-editor/vscode-extension build`），再 F5 启动
- **Webview 代码修改**：重新构建 webview（`pnpm --filter @mermaid-editor/vscode-extension build:webview`），在扩展宿主中 `Ctrl+R` 刷新

## 5. TRAE 配置

在 TRAE 的 MCP 服务器配置中添加：

```json
{
  "mcpServers": {
    "mermaid-editor": {
      "url": "http://localhost:14514/mcp",
      "transport": "streamable-http"
    }
  }
}
```

## 6. SKILL 安装

### 6.1 项目级安装（推荐）

```bash
# Windows (PowerShell)
New-Item -ItemType Directory -Force -Path .trae\skills\mermaid-flow-editor
Copy-Item -Path skills\mermaid-flow-editor\* -Destination .trae\skills\mermaid-flow-editor\ -Recurse

# Linux/macOS
mkdir -p .trae/skills/mermaid-flow-editor
cp -r skills/mermaid-flow-editor/* .trae/skills/mermaid-flow-editor/
```

### 6.2 用户级安装（全局可用）

```bash
# Windows (PowerShell)
Copy-Item -Path skills\mermaid-flow-editor -Destination $env:USERPROFILE\.trae\skills\ -Recurse

# Linux/macOS
cp -r skills/mermaid-flow-editor ~/.trae/skills/
```

### 6.3 验证安装

1. 重启 TRAE
2. 在对话中输入"看看我画的流程图"
3. AI 应该调用 get_input 工具读取画布内容

## 7. Web 编辑器（备选）

如果不使用 VSCode 插件，可通过 Web 编辑器使用：

```bash
# 启动 MCP 服务端
pnpm --filter @mermaid-editor/server start

# 启动 Web 编辑器（热重载）
pnpm --filter @mermaid-editor/web-editor dev
```

访问 http://localhost:5173 ，在画布上绘制流程图。

## 8. 使用方式

### 8.1 场景一：用户画图 → AI 分析

1. 打开 VSCode 插件编辑器或 Web 编辑器
2. 在画布上绘制流程图
3. 在 TRAE 对话中说"看看我画的流程图"或调用 `/mermaid-flow-editor`
4. AI 自动调用 get_input 读取画布并分析

### 8.2 场景二：AI 展示流程 → 用户查看

1. 在 TRAE 对话中询问复杂流程（如"解释用户登录流程"）
2. AI 判断是否调用 create_view 展示到画布
3. 在编辑器中查看可视化流程图，可进一步编辑

## 9. 故障排查

### 9.1 MCP 服务连接失败

```bash
# 检查 MCP 服务是否启动
curl http://localhost:14514/health

# 检查端口是否被占用
# Linux/macOS
lsof -i :14514
# Windows (PowerShell)
netstat -ano | findstr :14514
```

### 9.2 VSCode 插件问题

- **Webview 未构建**：运行 `pnpm --filter @mermaid-editor/vscode-extension build:webview`
- **连接状态一直"重连中"**：查看"Mermaid Editor Server"输出面板日志，可能是 pnpm 不在 PATH 中
- **服务端 20s 内未就绪**：查看输出面板日志，可手动运行 `pnpm --filter @mermaid-editor/server start` 验证

### 9.3 SKILL 未触发

- 检查 SKILL 文件位置是否正确（.trae/skills/mermaid-flow-editor/SKILL.md）
- 检查 frontmatter 的 description 字段是否包含触发关键词
- 重启 TRAE

### 9.4 get_input 返回"画布已消费"

- 在编辑器中点击"重新启用"按钮
- 或编辑画布（编辑后自动重置为待消费）
- 重新询问 AI

### 9.5 WebSocket 连接失败

- 检查浏览器控制台是否有 WebSocket 错误
- 确认服务已启动且端口 14514 可访问
- 检查防火墙设置

## 10. 卸载

```bash
# Windows (PowerShell)
Remove-Item -Recurse -Force .trae\skills\mermaid-flow-editor

# Linux/macOS
rm -rf .trae/skills/mermaid-flow-editor

# 移除 TRAE 的 MCP 配置（删除 mermaid-editor 条目）

# 停止 MCP 服务（Ctrl+C）

# 移除项目（可选）
# Windows (PowerShell)
Remove-Item -Recurse -Force mermaid2ai-chat
# Linux/macOS
rm -rf mermaid2ai-chat
```
